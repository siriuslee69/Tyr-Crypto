What this shell is for

One loader, many algorithm notes

This root page loads standalone algorithm documents from the corresponding subfolders under docs/. The idea is that every algorithm can keep its own detailed writeup, while the shell provides a common navigation layout, typography, and LaTeX rendering surface.

Conventions used by these pages

  • Start high-level, then descend into modules, then into hot loops.
  • Separate speed work, security work, and rollback decisions explicitly.
  • Use small numeric examples that a reader can recompute by hand.
  • Keep paper references tied to the exact implementation choices they influenced.
